# Knowledge System — Civilization Knowledge System

**Status:** Draft  
**Discussion:** Open

A digital civilization that evolves needs not only internal rules but also **continuous input of ideas**. Otherwise it becomes a closed system. This document describes the **sources of information** and how they feed into the project.

---

## Three Sources of Knowledge

| Source | Examples | Role |
|--------|----------|------|
| **Human Discussion** | GitHub Issues, X (Twitter), forums, community | New ideas, questions, institutional proposals |
| **Agent Reasoning** | Debate, simulation, proposals, rule revision inside the world | AI-generated governance ideas |
| **External Knowledge** | Research papers, history, economics, philosophy, political theory | e.g. Confucian thought, social science, AI governance research |

---

## Idea Pipeline（思想采集管道）

Flow:

```
X / GitHub / Research
        ↓
Idea Collection
        ↓
Semantic Structuring
        ↓
Community Discussion
        ↓
Agent Simulation
        ↓
Governance Proposal
```

**First discuss → then structure → then experiment.**

---

## Role of GitHub

GitHub acts as **Civilization Laboratory**:

- **Issues** — problems, new ideas, governance proposals (e.g. *Issue #21: Proposal: Reputation Governance System*)
- **Discussions** — open discussion, philosophical questions, long-term topics
- **Pull Request** — rule changes, protocol updates, code implementation

---

## Role of X (Twitter)

X serves as **Idea Radar** — to quickly surface:

- AI research, social debate, philosophical views, future tech  
- Topics such as: AI governance, digital identity, agent economy, open civilization  

Valuable content can be **collected and turned into Issues**.

---

## Idea Distillation（思想蒸馏）

```
Raw discussion
    ↓
Structured summary
    ↓
Ontology mapping
    ↓
Governance proposal
```

Example: from X, *"AI agents should vote based on reputation"* → distill into *Issue #35: Proposal: Reputation-based voting*.

---

## Open Civilization Design

The project should adhere to **Open Civilization Design**: anyone, any agent, can propose institutions. Acceptance is decided through **discussion**, **simulation**, and **governance**.

---

## Future: Research Agent

A **Research Agent** could read X, GitHub, and papers, and produce e.g. a **Digital Civilization Weekly Report** — top discussions on agent governance, semantic ontology, digital trust systems, etc.
